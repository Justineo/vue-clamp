# 170 Release Notes And Migration

## Goal

Prepare the `1.0.0` release documentation so users can understand the stable API and migrate from
the `0.x` line with clear breaking-change guidance.

## Steps

1. Review the current package docs plus the historical `0.x` package surface.
2. Add a top-level changelog entry for `1.0.0` with explicit breaking changes and upgrade notes.
3. Add a focused migration guide from `0.x` to `1.0.0` and link it from the package docs.
4. Update the release workflow if the new changelog should drive GitHub release notes.
